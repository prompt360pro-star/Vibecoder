1.1 — SUMÁRIO EXECUTIVO
text

PROBLEMA:
Milhões de pessoas querem aprender a programar com IA
(vibe coding), mas não existe um método estruturado,
gamificado e prático para aprender do zero ao expert.
As ferramentas existentes (Codecademy, freeCodeCamp,
Sololearn) ensinam programação TRADICIONAL, não a nova
realidade de desenvolvimento assistido por IA.

SOLUÇÃO:
VibeCode é um aplicativo mobile que ensina vibe coding
do básico ao expert através de um mentor IA personalizado
(Vi), missões interativas, projetos reais com deploy, e
gamificação profunda. O aluno aprende CONSTRUINDO apps
reais desde o dia 1.

PROPOSTA DE VALOR ÚNICA:
"Aprenda a construir apps com IA construindo apps com IA."
O app usa vibe coding como MÉTODO para ensinar vibe coding.
É meta, prático e viciante.

MERCADO:
- TAM: $50B (EdTech global)
- SAM: $8B (Coding education)
- SOM: $200M (AI-assisted coding education)
- Crescimento: 35% YoY no segmento IA

TARGET:
- Pessoas de 18-45 anos
- Sem experiência em programação OU programadores 
  querendo aprender IA-assisted development
- Empreendedores querendo criar seus próprios produtos
- Profissionais em transição de carreira

MODELO DE NEGÓCIO:
- Freemium com assinatura PRO ($14.99/mês)
- Plano anual ($99/ano)
- Lifetime ($299)
- Team ($29.99/mês por pessoa)

1.4 — PRINCÍPIOS DE DESIGN DO PRODUTO
text

1. CONSTRUA DESDE O DIA 1
   O aluno nunca fica mais de 10 minutos sem
   FAZER algo prático. Teoria é veículo, não destino.

2. FALHAR É APRENDER
   Erros são celebrados como oportunidade.
   Vi nunca julga. Vi guia.

3. PROGRESSÃO VISÍVEL
   O aluno SENTE seu progresso a cada sessão.
   XP, streak, badges, projetos no portfolio.

4. ADAPTATIVO, NÃO GENÉRICO
   O app se adapta ao ritmo, estilo e objetivo
   de cada aluno. Dois alunos nunca têm a mesma
   experiência exata.

5. REAL, NÃO SIMULADO
   Os projetos vão para a internet real.
   O portfolio é um site real. Os pagamentos
   são reais. Nada de "faz de conta".

6. MOBILE-FIRST, ANYWHERE
   Aprenda no ônibus, no sofá, na fila.
   Código longo é no desktop, mas conceitos
   e prática cabem no bolso.

7. SOCIAL BY DESIGN
   Aprender sozinho é difícil. Aprender com
   comunidade é sustentável. Co-op, ranking,
   ajuda mútua.

8. RESPEITE O TEMPO
   Se o aluno tem 10 minutos, dê valor em 10min.
   Se tem 2 horas, não bloqueie o fluxo.
   Sessões de 5min a 2h são igualmente válidas.

```

├── Offline: core features work offline
├── Bundle size: < 30MB initial download
├── Memory: < 200MB active usage
└── Battery: < 5% per hour of active use

API:
├── P50 latency: < 200ms
├── P95 latency: < 500ms
├── P99 latency: < 1000ms
├── Uptime: 99.9% (8.76h downtime/year max)
├── Throughput: 10,000 req/min sustained
└── Error rate: < 0.1%

AI:
├── Vi response (text): < 2 seconds TTFB
├── Vi response (voice): < 3 seconds
├── Code generation: < 5 seconds
├── Sandbox execution: < 3 seconds
├── Scan & Learn: < 4 seconds
└── Model router decision: < 100ms

DATABASE:
├── Query P95: < 50ms
├── Write P95: < 100ms
├── Connection pool: 20-100 connections
├── Read replicas: 2 minimum
├── Backup: every 6 hours
└── Point-in-time recovery: 30 days
```

### 5.2 — Segurança

```
AUTH:
├── Clerk managed authentication
├── MFA optional for all users
├── Session timeout: 30 days (mobile), 7 days (web)
├── Token rotation: every 24 hours
└── Brute force protection: 5 attempts → 15min lockout

DATA:
├── Encryption at rest: AES-256
├── Encryption in transit: TLS 1.3
├── PII minimization
├── Data residency: user choice (US/EU)
├── GDPR compliance: data export, deletion
├── LGPD compliance: same as GDPR
└── SOC2 Type II: target for Year 2

API:
├── Rate limiting: per user + per IP
├── Input validation: Zod schemas on all endpoints
├── Output sanitization: no PII in error messages
├── CORS: whitelist only known origins
├── CSP: strict content security policy
├── API keys: hashed, rotatable
└── Webhook signatures: HMAC-SHA256

SECRETS:
├── .env for local development
├── Vercel/Railway environment variables for production
├── No secrets in code or logs
├── Key rotation: every 90 days
├── Audit log: all secret access
└── Alert: on unusual access patterns

INFRASTRUCTURE:
├── DDoS protection: CloudFlare
├── WAF: CloudFlare rules
├── Dependency scanning: Snyk + Dependabot
├── Container scanning: if Docker used
├── Penetration testing: annually
└── Bug bounty: after 50K users
```

### 5.3 — Acessibilidade (a11y)

```
STANDARDS:
├── WCAG 2.1 Level AA compliance
├── Screen reader support (VoiceOver, TalkBack)
├── Minimum contrast ratio: 4.5:1 (text), 3:1 (large)
├── Touch targets: minimum 44x44px
├── Focus indicators: visible on all interactive elements
├── Semantic HTML in web views
├── Alt text on all images
├── Captions on all videos
├── Reduced motion respect (prefers-reduced-motion)
├── Dynamic type support (iOS/Android)
├── Color-blind safe (no color-only information)
└── Keyboard navigation (web app)
```

### 5.4 — Privacidade

```
DATA COLLECTION:
├── Minimum data principle
├── Analytics: anonymized by default (PostHog)
├── No selling of user data
├── No third-party ad tracking
├── Conversation with Vi: not used for AI training

├── Clear privacy policy: plain language
